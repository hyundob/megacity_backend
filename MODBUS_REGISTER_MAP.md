# Modbus Register Map (v2)

본 문서는 백엔드 DTO/엔티티 구조(`AREA_GRP_CD`, `AREA_GRP_ID`, `TM`, `CRTN_TM`, `FCST_TM`)에 맞춰 Group 매핑과 레지스터 주소를 정리한 기준안이다.

## 1) NULL 예약값 규칙
- FLOAT32: `NaN (0x7FC00000)`
- UINT16: `0xFFFF`
- UINT32: `0xFFFFFFFF`

## 2) Group 매핑 기준
- Modbus 주소 계산은 연속 인덱스(`groupIndex`)를 사용한다.
- DB 조회/매핑은 `area_grp_cd`, `area_grp_id`를 기준으로 수행한다.

## 3) 주소 배치 (충돌 없는 영역)

### 3.1 공통 상태 (고정)
- 업데이트 카운터: `30001` (size 2, UINT32)
- 예비: `30003` (size 98)
- 범위: `30001 ~ 30100`

### 3.2 그룹별 수소 실제 생산량
- 시작 주소: `30101`
- 블록 크기: `10`
- 공식: `addr = 30101 + (groupIndex - 1) * 10 + offset`
- 항목
  - 측정시간: `+0` (2, UINT32, `yyyyMMddHHmm -> epoch sec`)
  - 설비용량: `+2` (2, FLOAT32)
  - 수소생산량: `+4` (2, FLOAT32)
  - 예비: `+6` (4)

### 3.3 그룹별 수소 예측 생산량
- 시작 주소: `30501`
- 블록 크기: `590`
- 그룹 시작 공식: `groupBase = 30501 + (groupIndex - 1) * 590`
- 그룹 공통
  - 생성시간: `groupBase + 0` (2, UINT32)
- 선행시간(0~48, 49개)
  - 선행 시작: `groupBase + 2`
  - 선행 블록 크기: `12`
  - 공식: `leadBase = groupBase + 2 + leadIndex * 12`
  - 항목
    - 예측시간: `+0` (2, UINT32)
    - 선행시간: `+2` (1, UINT16)
    - 예측 설비용량: `+3` (2, FLOAT32)
    - 최종 생산량: `+5` (2, FLOAT32)
    - 예비: `+7` (5)

### 3.4 제주 계통 운영 데이터 (고정)
- 시작 주소: `53001` (기존 37001 영역과 충돌 방지)
- 항목
  - 측정시간 53001 (2, UINT32)
  - 현재수요 53003 (2, FLOAT32)
  - 태양광합계 53005 (2, FLOAT32)
  - 풍력합계 53007 (2, FLOAT32)
  - 신재생합계 53009 (2, FLOAT32)
  - 공급능력 53011 (2, FLOAT32)
  - 예비 53013 (8)

### 3.5 제주 예측 출력 제어 (고정)
- 시작 주소: `53021`
- 생성시간 53021 (2, UINT32)
- 예측시간 53023 (2, UINT32)
- 출력제어량 53025 (2, FLOAT32)
- 최소출력량 53027 (2, FLOAT32)
- 예비 53029 (12)

### 3.6 일간 수요 예측 (고정)
- 시작 주소: `53041`
- 생성시간 53041 (2, UINT32)
- 예측시간 53043 (2, UINT32)
- 수요예측량1~6: 53045~53055
- 최종/최소/최대: 53057~53061
- 예비: 53063 (8)

### 3.7 일간 신재생 예측 발전 (고정)
- 시작 주소: `53071`
- 공용 헤더
  - 생성시간 53071 (2, UINT32)
  - 신재생코드 53073 (1, UINT16)
  - 거래구분 53074 (1, UINT16)
- 선행시간 블록
  - 시작: `53075`
  - 블록 크기: `23`
  - 공식: `leadBase = 53075 + leadIndex * 23` (`leadIndex=0..48`)
  - 항목: 예측시간/선행시간/설비용량/예측량1~6/최종/최소/최대
- 예비: `54202` (9)

## 4) 소스 테이블
- `REP_DATA_HG_MEAS_GEM_GENT_DA` (수소 실제 생산량)
- `REP_DATA_HG_FCST_GEN_GENT_DA` (수소 예측 생산량)
- `REP_DATA_HG_FCST_NWP_DA` (기상 예측)

## 5) 구현 메모
- 시간 문자열(`yyyyMMddHHmm`)은 Modbus 전송 전 epoch seconds로 변환.
- FLOAT32/UINT32의 워드 순서(endianness)는 시스템 공통 규약으로 별도 고정.
- groupIndex는 외부 연동용 고정 인덱스이며 DB key 변경과 분리해 관리.
